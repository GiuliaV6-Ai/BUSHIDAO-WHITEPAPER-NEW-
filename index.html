アルゴリズム疑似コード
Algorithm Pseudocode — Quality Scoring Pipeline

Proof of Gi スコアリングパイプラインの核心。メッセージを受け取り、5次元を評価し、ポイントを付与するまでの全処理フローです。

メインスコアリング関数

function evaluate_message(message, member_id, channel_id): # ── ステップ 1: 前処理 ────────────────────────────── tokens = tokenize(message.text) lang = detect_language(tokens) patterns = extract_patterns(tokens) context = load_context(member_id, channel_id) # ── ステップ 2: 疑わしいパターンの事前チェック ────── if is_suspicious(patterns, context): log_suspicious_activity(member_id, patterns) return PointResult(points=0, reason="suspicious_pattern") # ── ステップ 3: 5次元スコアリング ─────────────────── d1 = score_language_appropriateness(lang, tokens) # 20% d2 = score_greeting_politeness(tokens, patterns) # 15% d3 = score_substance_value(tokens, context) # 40% ← 最重要 d4 = score_security_awareness(tokens) # 15% d5 = score_conversation_depth(tokens, context) # 10% # ── ステップ 4: 加重合算 ──────────────────────────── quality = (d1 * 0.20) + (d2 * 0.15) + (d3 * 0.40) + (d4 * 0.15) + (d5 * 0.10) # ── ステップ 5: 品質閾値チェック ──────────────────── if quality < QUALITY_THRESHOLD: # 6.0 return PointResult(points=0, reason="below_threshold", score=quality) # ── ステップ 6: ポイント計算（最大 25pt）─────────── base_points = int((quality / 10.0) * MAX_POINTS) # MAX_POINTS = 25 # ── ステップ 7: セキュリティボーナス ──────────────── bonus = calculate_security_bonus(tokens) total = base_points + bonus # ── ステップ 8: 永続化 ────────────────────────────── save_to_db(member_id, total, quality, channel_id) update_leaderboard(member_id, total) return PointResult(points=total, quality=quality, breakdown={ "d1": d1, "d2": d2, "d3": d3, "d4": d4, "d5": d5, "bonus": bonus })

次元 3「実質的貢献」詳細スコアリング

function score_substance_value(tokens, context): score = 0.0 # ─ 長さ・構造チェック ───────────────────────────── word_count = len(tokens) if word_count >= 100: score += 3.0 # 長文・詳述 elif word_count >= 40: score += 1.5 # 中程度 elif word_count >= 15: score += 0.5 # 短文 else: score -= 1.0 # 極短（スパムリスク） # ─ 専門用語・技術キーワード ────────────────────── tech_terms = count_domain_keywords(tokens, TECH_KEYWORD_LIST) score += min(tech_terms * 0.4, 2.0) # 上限 +2.0 # ─ 質問・提案パターン ──────────────────────────── if has_genuine_question(tokens): score += 1.0 if has_proposal(tokens): score += 1.5 if has_analysis(tokens): score += 2.0 # ─ ネガティブ要素 ──────────────────────────────── if is_pure_emoji_reaction(tokens): score = -3.0 # 強制上書き if is_copy_paste_repeat(tokens, context): score -= 2.0 return clamp(score, -3.0, 10.0)
