BushiDAO（ブシダオ）は、ブロックチェーン上での貢献を「義（Righteousness / Honor）」として定量化する初めての DAO ガバナンスシステムです。単なるポイント付与を超え、コミュニティの真の価値と持続可能性を高めるための技術的インフラを提供します。

本白書では、武士道哲学に根ざした貢献評価フレームワークを詳述し、Proof of Gi（義の証明）の技術実装を完全に開示します。5 つの次元（言語適切性・挨拶の質・貢献内容・セキュリティ意識・対話の深さ）での評価を実証します。

「寺子屋（テラコヤ）」の精神に基づき、すべてのメンバーが互いに学び合い、成長し続ける場を構築します。🏯

Section 01

BushiDAOの理念と使命

THE PHILOSOPHY & MISSION

Web3 ガバナンスの従来モデルは、シンプルなトークン投票から、二次投票や評判加重プロトコルへと進化してきました。しかし技術的な洗練さにもかかわらず、根本的な問いが残されています：

核心的問い

「真に意味のある貢献とは、何か？」🤔

この問いから生まれたのが、Proof of Gi（義の証明）です。武士道（Bushido）の視点を通じて、オンチェーン上の貢献を定量化する先駆的なアプローチです。

🌟 ミッション・ステートメント

BushiDAO は、次の原則に基づく持続可能なコミュニティ・ガバナンスを実現します：

  • オンチェーンの行動が、その文脈的な影響によって公正に評価される
  • メンバーが透明な相互評価システムを通じて成長できる
  • 貢献が時間をかけて複利的に積み上がる（定期リセットなし）
  • セキュリティ優先の設計がすべての参加者を守る
  • 日本の文化的価値観がブロックチェーン技術とシームレスに融合する

🏗️ 技術コンポーネント

1. データ収集レイヤー

Discord のコミュニティチャンネルからメッセージを収集します（プライベート DM へのアクセスは一切なし）。パターンマッチングによるスレッド紐付けとチャンネル分類を行い、セキュリティフィルターで認証情報の露出を検出します。

2. 品質評価エンジン

5 次元加重スコアリング：

PYTHON quality_score = ( language_appropriateness * 0.20 + # コミュニティ言語設定との一致 greeting_politeness * 0.15 + # 単純な「こんにちは」を超えた挨拶 substance_value * 0.40 + # [⭐ 最重要] 実質的な貢献の質 security_awareness * 0.15 + # セキュリティ警告へのボーナス conversation_depth * 0.10 # 双方向対話 vs 一方的発信 ) final_points = int((quality_score / 10) * 25) # 最大 25pt にキャップ # 🚨 品質ゲート — 最重要！ IF quality_score >= 6.0 AND NOT suspicious_pattern(): ポイント付与（quality_score に比例） ELSE: ポイント付与なし（品質閾値未達）

Section 04

パフォーマンス評価基準

FIVE-DIMENSION QUALITY FRAMEWORK — 五つの次元の質的判断
⚠️ 品質閾値ルール

最終品質閾値：5 つの次元の合計スコアが 6.0 / 10 以上でなければ、いかなるポイントも付与されません。この閾値がスパム・自動挨拶ファーミング・ゲーミングを防ぎ、真のコミュニティ参加に報酬を与える要です。

20%比重

Section 05

技術的実装詳細

PROOF OF GI — TECHNICAL ARCHITECTURE

🧪 技術的検証

既存の scoring_system.py コードと評価ルーブリックに基づく、検証済みアーキテクチャは以下のとおりです：

コア数式 quality_score = lang*0.20 + greeting*0.15 + substance*0.40 + security*0.15 + engagement*0.10 IF final_quality >= 6.0 AND not_suspicious_pattern(): awarded_points = int((final_quality/10)*25) # 最大 25 pt

✅ アーキテクチャ設計（コードレビュー検証済み）

  • 5 次元評価エンジン（多次元品質スコアリング）
  • パターンベースのアンチゲーミング検出
  • ローカル SQLite ＋ セッションメモリの永続化
  • チャンネル観察設定

🎯 Proof of Gi の核心設計原則

1️⃣ 品質ゲート制報酬 ⭐ 最重要

空虚な貢献は自動的に拒否されます。真に思慮深いコンテンツのみがポイントの対象になります。≥6.0 品質ゲートがアンチゲーミングの中核機構です。

2️⃣ 複利成長モデル 📈

ポイントはリセットや失効がありません。貢献 #1（+5 pt）＋ 貢献 #10（+8 pt）＋ 貢献 #24（+12 pt）→ 累計 37 pt！ 一つひとつの積み重ねが長期的な報酬と評価につながります。

3️⃣ セキュリティ・ファーストのコミュニティ 🔒

セキュリティリスクを公開で警告したメンバーにはボーナスポイント（+5 / +10）が付与されます。空虚な称賛よりも誠実な対話へのインセンティブを整合させます。

4️⃣ 透明性と説明責任 📖

すべての評価は明示的なルーブリック、検証可能な基準、スパムを報酬前に排除する品質ゲートを使用します。スコアリング数式は全メンバーに公開されています。

5️⃣ 文化的共鳴 🏯

日本の価値観が分散型ガバナンスへシームレスに変換されます。武士道の名誉規範 → Proof of Gi の定量化。寺子屋精神（学び合うコミュニティ）が受け継がれます。

セキュリティ＆安全プロトコル

武士の戒め — SECURITY GUARDIANSHIP

🛡️ セキュリティアーキテクチャ概要

レイヤー保護メカニズム軽減される脅威

🔐 多層セキュリティモデル

レイヤー 1 — アクセス制御

重要ルール：DM は絶対に使用しません！
✅ 許可：チャンネル投稿・スレッド返信・コミュニティカテゴリ内の @メンション
❌ ブロック：DM 経由での問い合わせ・応答の試みは自動的に拒否

レイヤー 2 — パターンベースのアンチゲーミング

疑わしいパターンを識別します：過剰な装飾スパム（+++++・*****）、繰り返しの定型文、60 秒以内に複数チャンネルへの同一メッセージ、対話なしの空虚な称賛。

レイヤー 3 — 認証情報露出検出（最高優先度）

秘密鍵・シードフレーズ・API トークンにゼロトレランスを適用。64 文字の 16 進数文字列・12 / 24 語のニーモニックフレーズ・露出した API キーをパターンマッチングで検出。検出時はメッセージを即時拒否します。

レイヤー 4 — データ永続化セキュリティ

すべてのデータはローカルにのみ保存されます。✅ ローカル SQLite（ファイルベース・外部同期なし）。✅ セッションメモリバックアップ。❌ AWS S3 不使用。❌ Firebase / MongoDB 不使用。
